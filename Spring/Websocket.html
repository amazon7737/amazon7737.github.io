    # Websocket 에 관하여

    스프링 공식문서에는 Websocket 구현이 STMOP js , SOCK js 기반으로 작성을 해두었다. 하지만, 다른 Web framework(React) 를 활용하여 개발을 진행할 경우에는 해당 라이브러리에 종속적이게 되어 세부적인 기능 개발에 문제가 발생한다. 이를 보완하기 위하여 조금 조사를 해보았다.

    참고 : https://spring.io/guides/gs/messaging-stomp-websocket

    # WwbSocket 클래스

    
@Service
@ServerEndpoint("/socket/chatt")
public class WebSocketChat {
  private static Set<Session> clients = Collections.synchronizedSet(new HashSet<Session>());
  private static Logger logger = LoggerFactory.getLogger(WebSocketChat.class);

  @OnOpen
  public void onOpen(Session session){
    logger.info("open session : {}, clients = {}", session.toString(), clients);
    Map<String, List<String>> res = session.getRequestParameterMap();
    logger.info("res = {}", res);

    if(!clients.contains(session)){
      clients.add(session);
      logger.info("session open : {}", session);
    }else{
      logger.info("이미 연결된 session");
    }
  }

  @OnMessage
  public void onMessage(String message, Session session) throws IOException{
      logger.info("receive message : {}", message);

      for(Session s : clients){
        logger.info("send data : {}", message);
        s.getBasicRemote().sendText(message);
      }
  }

  @OnClose
  public void onClose(Session session){
    logger.info("session close : {}", session);
    clients.remove(session);
  }


}

    어떤 라이브러리에도 종속적이지 않은 javax 의 websocket 자체 라이브러리이다. js에 종속적인 모습을 보이면 오히려 웹프레임워크를 사용할 이유가 없고 단순한 페이지 전환만이 일어난다. 하지만, endpoint 를 지정하여 자바에서 직접 로직을 처리하고 restcontroller 로 리턴을 준다면 서비스를 만드는데 도움이 될 것이다.

    추가적인 문제점이 발생하면 다시 기록해봐야겠다.
